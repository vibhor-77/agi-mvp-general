ARCHITECTURE
================================================================================

SEARCH PIPELINE INTEGRATION (Step 3.95):

  solve_task()
    ├─ Step 1-3: Single primitives, culture, pair exhaustion
    ├─ Step 3.5: Triple search
    ├─ Step 3.9: Object-centric reasoning
    ├─ Step 3.95: CELL RULE SYNTHESIS (NEW)
    │   ├─ Enumerate border coloring: O(|colors|)
    │   ├─ Enumerate swaps: O(|colors|²)
    │   ├─ Enumerate fill-from-neighbors: O(|colors|)
    │   └─ Return best candidate (score ≥0.85)
    └─ Step 4+: Evolution with cell rule seeds

ENUMERATION STRATEGIES:

  1. Border Coloring
     - For each color in input ∪ output
     - Create rule: if is_border() then set_color(c)

  2. Color Swaps
     - For each from_color in input, to_color in output
     - Create rule: if is_color(from) then set_color(to)

  3. Fill from Neighbors
     - For each (target, source) color pair
     - Create rule: if is_color(target) then copy_neighbor_matching(source)

COMPLEXITY:
  - Time: O(|candidates| × |examples| × |grid_size|)
  - Space: O(|colors|²)
  - Typical: 20-40 candidates, <100ms evaluation
